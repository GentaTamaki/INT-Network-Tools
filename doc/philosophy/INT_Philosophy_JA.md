

# INT Network Tools
## 設計思想 (Philosophy)

INT Network Tools は、現代のソフトウェア中心の制作環境と、従来のハードウェアベースのタイミングシステムを橋渡しすることを目的として設計されています。

映像制作、放送、舞台、バーチャルプロダクションなどの現場では、長年にわたり **タイムコードによる同期** が重要な役割を果たしてきました。しかし近年、制作環境は大きく変化し、多くのツールがソフトウェア中心のワークフローへと移行しています。

INT Network Tools は、この変化する環境の中で **ネットワークを利用したタイムコード配信基盤** を提供します。

---

# 1. ソフトウェアとハードウェアの橋渡し

現代の制作ツールの多くはソフトウェア環境で動作します。

例：

- NLE（DaVinci Resolve など）
- Unreal Engine
- メディアサーバー
- バーチャルプロダクションシステム

これらのシステムはネットワーク環境との親和性が高い一方、従来の同期インフラとは直接接続できないことが多くあります。

一方で、放送や舞台の現場では今でも **LTC（Linear Timecode）ベースの同期システム** が広く使われています。

例：

- 放送設備
- 舞台照明システム（DMX）
- メディア再生ハードウェア
- タイムコードディスプレイ
- 各種プロフェッショナル機器

INT Network Tools は、ネットワーク経由でタイムコードを配信し、必要に応じて **LTC 信号へ変換する Bridge デバイス** を介して、これらのシステムを接続します。

```
Software Systems
       ↓
      INTTC
       ↓
     Bridge
       ↓
       LTC
       ↓
Legacy Hardware
```

---

# 2. ネットワークによるタイムコード配信

INT Network Tools は既存の同期システムを置き換えることを目的としていません。

代わりに、既存のインフラの上に **ネットワークベースのタイムコード配信レイヤー** を追加します。

これにより次のような構成が可能になります。

- ソフトウェアタイムラインからタイムコード生成
- ネットワーク経由で複数デバイスへ配信
- Bridge デバイスによる LTC 変換
- 従来機器との互換性維持

この構造により、既存ワークフローを壊すことなく柔軟なシステム設計が可能になります。

---

# 3. 異なる同期要求への対応

制作環境によって、必要とされる同期精度は大きく異なります。

編集環境では次の要素が重視されます。

- 応答性
- 柔軟性
- 実装のシンプルさ

一方、次のような環境では **フレーム単位で安定した同期** が必要になります。

- バーチャルプロダクション
- LEDウォール
- Unreal Engine
- 舞台照明システム
- メディアサーバー

この違いに対応するため、INT Network Tools では **2種類の Output Policy** を定義しています。

```
Loose
Tight
```

### Loose

編集・モニタリング用途向け。

特徴：

- パケット駆動更新
- 小さなタイミング揺らぎを許容
- 再同期を高速に行う

用途例：

- クライアントモニター
- 編集レビュー
- ソフトウェア表示

### Tight

同期精度が重要な環境向け。

特徴：

- 連続したフレーム進行
- 安定したフレーム境界
- クロック制御された出力

用途例：

- Unreal Engine 同期
- LEDウォール再生
- 舞台照明システム
- メディアサーバー

---

# 4. ソフトウェア定義タイミング

INT Network Tools はタイムコード処理を **ソフトウェア定義システム** として扱います。

Receiver は内部的に時間を次の3層に分離して処理します。

```
Packet Time
Resolved Time
Output Time
```

この構造により、以下が可能になります。

- ネットワークジッター耐性
- 出力の連続性維持
- パケット途絶時の安定動作
- 異なる同期環境への適応

---

# 5. 互換性を最優先する設計

INT Network Tools の重要な設計原則は次の通りです。

> 既存ワークフローとの互換性を犠牲にしない

そのため、**LTC 出力は常に第一級のインターフェース** として扱われます。

LTC は現在もなお、

- 放送
- 舞台
- メディア再生
- 映像制作

など、多くの現場で使われている最も普遍的な同期信号です。

INT Network Tools はこのエコシステムと共存することを前提に設計されています。

---

# 6. 将来の同期システムへの基盤

INT Network Tools は特定の実装に限定されるものではありません。

将来的には次のような拡張を想定しています。

- PTP（Precision Time Protocol）同期
- GPS同期
- マルチ Sender 環境
- 分散同期システム

プロトコル伝送、時間解決、出力生成を分離することで、INT Network Tools は将来のタイミング技術の基盤として機能する設計になっています。
